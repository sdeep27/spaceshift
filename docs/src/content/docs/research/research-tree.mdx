title: Research Tree

`research_tree` is the flagship research function. It decomposes a topic into sub/super/side branches, then generates a titled response for every node in the tree. Outputs include citation URLs when web search is used.

```python
from spaceshift import research_tree

result = research_tree(
    "How do general-purpose technologies reshape economic structures?",
    sub_n=[5, 3],
    super_n=[5],
    side_n=[3],
)

print(result["root_output"])
print(len(result["outputs"]))
```

## Parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| `prompt` | required | The research question, or a pre-built tree dict from `prompt_tree` |
| `sub_n` | `[5, 3]` | Subprompt levels — `[5, 3]` = 5 subtopics, each split into 3 |
| `super_n` | `[5]` | Superprompt levels |
| `side_n` | `[3]` | Sideprompt levels |
| `model` | `1` | Model for tree construction (prompt generation) |
| `output_model` | same as `model` | Model for generating responses at each node |
| `search` | `"auto"` | Web search for output generation — auto-detects model support |
| `search_tree` | `"auto"` | Web search for tree construction prompts |
| `save` | `True` | `True` to auto-name from prompt, string for explicit path, `False` to skip |
| `concurrency` | `5` | Max parallel API calls |
| `v` | `True` | Verbose progress output |

## Output structure

Each entry in `result["outputs"]` contains:

```python
{
    "id": "sub_0",                    # node identifier
    "prompt": "...",                  # the research question for this node
    "response": "...",               # full markdown response
    "title": "Economic Disruption",  # LLM-generated title (used as filename)
    "direction": "sub",              # sub, super, side, or root
    "depth": 0,                      # tree depth level
    "parent": "...",                 # parent prompt text
    "citations": ["https://..."],    # URLs from web search (when available)
}
```

## File output

When `save` is enabled, each node is saved as a markdown file with YAML frontmatter. Files are named by their title (e.g., `Economic Disruption.md`) rather than node IDs, with automatic deduplication for collisions. A `tree.svg` visualization is also generated.

## Web search

Search defaults to `"auto"` — it probes whether the selected model supports web search and enables it automatically. No need to pass `search=True` unless you want to force it on (which will auto-switch to a search-capable model).

When search is active, citation URLs are extracted and saved in the YAML frontmatter of each output file.
